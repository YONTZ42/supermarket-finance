課題:スーパーマーケット決算データの一元管理
あなたは日本国内に複数店舗を展開するスーパーマーケットチェーンの情報システム担当です。
現在、各店舗では決算データを独自にExcelで管理していますが、管理形式や項目が店舗ごとに異なっ
ており、統一的に比較・集計できない状況です。
経営層は「半期(上期・下期)ごとの売上・経費・利益」を全店舗共通で把握したいと考えています。
そこで、Reactを用いたフロントエンドの管理画面を実装し、バラバラのデータを一元管理できる仕組みを
構築してください。
※なお、本課題は弊社内での実際の業務をイメージしているため、あらゆるAIツールを利用して構いま
せん。
(弊社ではむしろコーディングに生成AIを活用することを推奨しています)

店舗ごとの管理ルール
東京店舗
● 管理項目
売上:生鮮食品、加工食品、菓子、飲料、惣菜
経費:人件費、水道光熱費、広告宣伝費、物流費、その他経費
● 管理方法
○ 3, 6, 9, 12月の「累積データ」
○ 例:6月の値は1〜6月の累積、9月は1〜9月の累積

大阪店舗
● 管理項目
売上:菓子、飲料、おもちゃ、日用品
経費:人件費、広告宣伝費、物流費、その他経費、販管費
● 管理方法
○ 3, 6, 9, 12月の「期間別データ」
○ 例:6月の値は4〜6月、12月は10〜12月の期間のデータ

名古屋店舗
● 管理項目
売上:生鮮食品、菓子、飲料、惣菜、日用品
経費:人件費、広告宣伝費、物流費、その他経費、雑費
● 管理方法
○ 6, 12月の「累積データ」
○ 例:6月の値は1〜6月の累積、12月は1〜12月の累積

実装画面
データ登録画面
● 店舗を選んで、その店舗ごとの項目(売上・経費)を入力できる UI を実装すること
○ 東京は四半期ごとに、累積金額の入力
○ 大阪は四半期ごとに、対象期間内に発生した金額の入力
○ 名古屋は半期ごとに、累積金額の入力
● 入力されたデータを内部で統合し、半期ごとのサマリ画面で扱うデータに整形できること
● 店舗ごとのアクセス制御等は不要です
サマリ画面
● 半期(上期:1〜6月、下期:7〜12月)の「売上合計・経費合計・利益」を共通フォーマットで一覧化
● 累積データを扱う店舗(東京・名古屋)は、差分計算で期間ごとの実績に変換すること
● Excelの「ダッシュボード」シートに関数が組まれているので、ヒントにしてください

例:
年度 店舗 半期 売上合計 経費合計 利益
2022 東京 上期 ... ... ...
2022 大阪 上期 ... ... ...
2022 名古屋 上期 ... ... ...
2022 東京 下期 ... ... ...
2022 大阪 下期 ... ... ...
2022 名古屋 下期 ... ... ...

必須要件
1. React でフロントエンドを実装すること(Next.js等可)
2. 店舗ごとの管理画面、サマリ画面を実装すること
3. Excelのデータが入っている状態で起動できること
任意要件
● データの永続化ができること(バックエンドは任意)
● 拡張性を考慮し、店舗別の項目を追加できるようにすること
● 管理対象の店舗が増えた際に、専用のロジックを組む必要がないこと
● 設計意図や要件の理解について説明したドキュメント(日本語または英語)
○ 形式は自由(README、Markdown、Word、Google Docsなど)
○ 実装の意図や要件解釈が分かる補足資料があると望ましい

採点観点
● データ処理の正確さ
● データ入力画面のUXの良さ
● 任意要件の実装状況
